Security Issues

Potential security issues at this time include having unwanted users access the site and submit information. This can be solved with a log in system that allows users access to certain pages, and denies it to others. (This is kind of here, but not perfect). Another issue is SQL injection, where users may try to maliciously manipulate the database by submitting SQL statements. This is solved by binding the user input into the query via the "->bindParam" method, which adequately formats the input to not tamper with the form process. Another potential issue is HTML injection. This can be solved by calling the function htmlspecialcharacters() on the date before displaying it to the page (or submitting it to the database).
