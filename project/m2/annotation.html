Usage of PHP

  • For this milestone, PHP was primarily implemented submit bands and shows. Because there is a heavy reliance on a database, the website now just takes the form data and displays it on the appropriate page as though it were pulling from a database. The only way to see this page is to submit the form. Clicking on a link of a band will lead to a static, hardcoded page.
    • Adding a new band at newBand.php forwards to viewBand.php, displaying the recent input.
    • Adding a new show at newShow.php forwards to viewShow.php, displaying the recent input.
  • The start of a login system is included, but does not affect much at this time. Simply takes username "aquadome" and password "password".

Source code for all pages in the project

  • index.phps is the main landing page
  • login.phps form for user to log in
  • dispShows.phps displays a list of shows in the database (static)
  • dispBands.phps displays a list of bands in the database (static)
  • newShow.phps add new show form page
  • viewShow.phps displays newShow form input
  • newBand.phps add new band form page
  • viewBand.phps displays newBand form input
  • Included files for form processing (note: I am undecided if this is the best way of modularizing the displaying of pages and the processing of them, or if it may be better to have it all in one document. At this time, it seems that this separation allows for an easier use of the header() function in php, which cannot have any output come before it.
